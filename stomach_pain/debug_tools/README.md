# 调试工具说明

本目录包含开发过程中使用的调试插件,用于探索autMan的API结构。

## 文件说明

### 检测全局.js
检测autMan中可用的全局函数和对象。

**触发**: 发送 `检测全局`

**用途**: 
- 查看哪些API函数是全局可用的
- 确认bucketGet、bucketSet等函数是否存在

### 测试ID.js
测试如何获取senderID。

**触发**: 发送 `测试ID`

**用途**:
- 探索process.argv
- 检查全局变量
- 发现Sender不需要参数

### 测试Sender.js
测试Sender对象的使用方法。

**触发**: 发送 `测试Sender`

**用途**:
- 检查Sender的方法和属性
- 确认如何调用reply()

### 调试肚子疼.js
调试关键词匹配和消息内容。

**触发**: 发送 `肚子疼xxx`(任何包含肚子疼的消息)

**用途**:
- 查看收到的原始消息内容
- 测试关键词匹配是否正确
- 逐字符分析消息

## 使用建议

这些工具主要用于开发调试,正式使用时不需要部署到服务器。

如果您在开发其他autMan插件时遇到API问题,可以参考这些调试工具的代码。
