# 奥力给记录插件 💪

一个基于autMan开发的Telegram插件，用于记录每次拉屎的时间戳，帮助你追踪肠道健康。

## 功能特点 ✨

- 🕐 **自动记录时间** - 发送"奥力给"即可记录当前时间
- 📊 **时间轴视图** - 智能分页显示历史记录
- 📈 **统计分析** - 自动统计每日次数、月度摘要、平均频率
- 🎨 **频率标记** - 用不同颜色标记每日频率(🟢🟡🟠🔴)
- 💾 **数据持久化** - 所有记录安全存储，随时查询

## 使用说明 📖

### 基本命令

| 命令 | 功能 |
|------|------|
| `奥力给` | 记录一次拉屎时间 |
| `奥力给记录` | 查看所有历史记录(时间轴视图) |
| `清空奥力给记录` | 清空所有记录 |
| `奥力给帮助` | 显示帮助信息 |

### 使用示例

1. **记录拉屎时间**

## 安装方法 🔧

### 方式一: 直接复制文件

1. 将 `奥力给.js` 文件复制到autMan的插件目录
2. 在Telegram中发送 `/reload` 或重启autMan
3. 发送 `奥力给帮助` 测试插件是否正常工作

### 方式二: 使用Git克隆

```bash
cd /path/to/autman/plugins
git clone https://github.com/yourusername/aoligei-plugin.git
```

## 技术细节 🔍

- **存储桶**: `aoligei_record`
- **数据格式**: JSON格式存储，包含时间戳和格式化时间
- **用户隔离**: 每个用户的数据独立存储 (`user_{userID}`)
- **ES5兼容**: 使用ES5语法，适配autMan环境

## 数据隐私 🔒

- 所有数据仅存储在autMan的本地存储桶中
- 每个用户的数据完全隔离
- 可随时清空个人记录

## 常见问题 ❓

**Q: 插件不响应怎么办？**  
A: 检查插件是否正确安装，尝试发送 `/reload` 重新加载插件。

**Q: 可以修改关键词吗？**  
A: 可以，修改代码中的 `// [rule: .*奥力给.*]` 和相关关键词判断逻辑。

**Q: 数据会丢失吗？**  
A: 不会，所有数据持久化存储在autMan的存储桶中，除非手动清空。

**Q: 支持多用户吗？**  
A: 支持，每个用户的数据独立存储，互不影响。
